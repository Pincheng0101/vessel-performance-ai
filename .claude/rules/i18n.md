# i18n Wording Conventions

Applies to translation files in `i18n/locales/` (both `en.js` and `zh-TW.js`).

## English field labels (`__field*`) — Title Case

Capitalize:
- The first and last word, always.
- All content words: nouns, verbs, adjectives, adverbs, pronouns.
- Acronyms stay all-caps: `LLM`, `API`, `JSON`, `AWS`, `ID`, `ARN`, `MCP`,
  `URL`, `SQL`, `S3`, `IAM`, `HTTP`, `HTTPS`, `MFA`.
- Single-letter notation stays as-is: `Top K`, `Top P`.

Lowercase (Chicago Manual of Style):
- Articles: `a`, `an`, `the`.
- Coordinating conjunctions: `and`, `or`, `but`, `nor`.
- **All** prepositions regardless of length: `of`, `in`, `on`, `at`, `by`,
  `to`, `for`, `with`, `from`, `as`, `into`, `over`, `under`.

Hyphenated compounds — capitalize the first part only (unless the rest is a
proper noun): `Real-time`, `Built-in`, `Cross-region`.

Examples:

| ✅ | ❌ |
|---|---|
| `Day of Month` | `Day of month` / `Day Of Month` |
| `Maximum Items` | `Maximum items` |
| `Sites to Search` | `Sites To Search` |
| `Combine Original Input with Result` | `Combine Original Input With Result` |
| `Use Cached Connection` | `Use cached connection` |
| `Enable Real-time Streaming at Task End` | `Enable Real-Time Streaming At Task End` |

## English tooltip (`__tooltip*`) — verb prefix by category

Pick the verb that matches the tooltip's purpose, and use it consistently
across the whole category. Always use the imperative form (`Specify`, not
`Specifies`; `Define`, not `Defines`).

| Category | Use | Example |
|---|---|---|
| Input field (text / number / JSON / dropdown) | `Specify the …` | `Specify the Athena data catalog name.` |
| Toggle — describe behavior on enable | `When enabled, …` | `When enabled, queries are run in read-only mode.` |
| Toggle — grant a capability | `Allow …` | `Allow users to switch between multiple LLMs.` |
| Settings container (parent of multiple sub-settings) | `Configure …` | `Configure an OpenSearch tool the agent can use.` |
| Tool / workflow node function description | Functional verb (`Call` / `Invoke` / `Run` / `Generate` / `Read` / `Wrap` / `Pass` / `Apply` / `Transform`) | `Call LLM to generate a natural language response.` |
| Concept / parameter explanation (incl. formulas) | Noun phrase, or `X is …` / `X controls …` | `Top P controls randomness by selecting words …` |
| Read-only / state explanation (no input action) | Noun phrase | `The default LLM cannot be removed from the switchable LLM list.` |

Treat numeric limit / threshold / timeout fields as input fields — use
`Specify the maximum …`, not `Set …` or a bare noun phrase.

## Traditional Chinese tooltip (`__tooltip*` in `zh-TW.js`) — 動詞前綴對照

| 類型 | 開頭 | 範例 |
|---|---|---|
| 輸入欄位指引 | `指定…` | `指定 Athena 的資料目錄名稱。` |
| 開關行為描述 | `啟用後，…` | `啟用後，查詢會以唯讀模式執行以避免寫入。` |
| 開關權限授予 | `允許…` | `允許使用者在與 Agent 互動時切換使用多個大型語言模型。` |
| 設定容器 | `設定…` | `設定 Agent 可使用的 OpenSearch 工具…` |
| 工具/節點功能描述 | 功能動詞（呼叫／執行／讀取／生成／傳遞／套用） | `呼叫大型語言模型，生成自然語言回應。` |
| 概念/參數說明 | 名詞片語 | `Top P 透過選擇累積機率達 P 的詞彙來控制隨機性…` |
| 唯讀/狀態說明 | 名詞片語 | `預設大型語言模型不能從可切換的大型語言模型清單中移除。` |

數值上限／逾時／長度欄位視為輸入欄位，用「指定…」開頭，不要寫成「設定…」
或無動詞名詞片語。

## Other conventions

- **Token 大小寫**：
  - zh-TW 在中文文案中一律寫 `Token`（首字大寫），不要寫 `token`。例如 `總 Token 數`、`消耗更多 Token`。
  - 英文遵循標準句法：作為 Title Case label 時大寫（`Total Tokens`、`Input Tokens`），在句子中作為一般名詞時小寫（`token usage`、`fewer tokens`）。
- **「大型語言模型」 vs `LLM` (zh-TW)**：中文文案統一用「大型語言模型」，避免縮寫成 `LLM`。例外：縮寫無對應中文展開時保留原樣（`MCP 工具`、`MCP 伺服器`）。
- **句尾標點**：tooltip、instruction、subtitle 的句子皆以 `.`（英）或 `。`（中）收尾。短句片段（如 `__hint*`、`__instructionWorkflowWaitInterval`）可不收尾。
- **第三人稱動詞**：避免 `Specifies` / `Sets` / `Defines` / `Refers to` / `Describes` 起頭，一律改成祈使句（`Specify` / `Set` / `Define` / `Describe`）。
