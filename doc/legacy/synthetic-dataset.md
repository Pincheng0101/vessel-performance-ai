# Synthetic Raw Dataset (POC M1)

5-year, physics-consistent **raw-zone** maritime data for a 9-vessel container
fleet. Generated by `ym_datalake/synthetic_data/`, landed in S3 as JSONL,
catalogued in Glue, queried via Athena. This is the M1 deliverable: raw fields
complete, data passes the ¬ß3.2 **C1‚ÄìC17** consistency checks, YM WELLNESS
storyline ready. M2 ETL (ISO 15016/19030) consumes it and must recover the
injected speed loss.

- Generator package: `ym_datalake/synthetic_data/` (numpy-only, no pandas).
- Curve module `curves.py` is the **single shared speed-power source of truth**
  reused by M2 ‚Äî sharing it is what makes the injected speed loss recoverable.

## 5. Generation logic & C1‚ÄìC17 mapping

Per vessel √ó day the forward model derives values in the order that guarantees
recoverability (see `generate.py::_forward_day`). `s` = the fractional speed loss
at constant power; `V_ref = STW/(1‚àís)` is the clean-equivalent speed.

1. **Ground truth first** ‚Äî `days_since_cleaning`, true `s` (fouling_rate¬∑days +
   bounded noise), reset at events (**C9**).
2. Loading (laden/ballast) ‚Üí cargo ‚Üí **Œî from mass balance** (lightship + cargo +
   ballast + fuel + stores) ‚Üí invert to `mean_draft` via one hydrostatic curve ‚Üí
   fore/aft/trim (**C5**, **C6**).
3. Seasonal met-ocean: Beaufort drives wind & wave; density falls with temp
   (**C11**).
4. **Command STW** near design speed (slow-steaming factor + variation).
5. `V_ref = STW/(1‚àís)`; **`P_corrected = a¬∑V_ref^n¬∑(Œî/Œî_ref)^(2/3)`** (**C1** ‚Äî
   the exact relation M2 inverts).
6. Added resistance `R_AA` (Blendermann wind, A_XV) + `R_AW` (STAWAVE-1
   `‚âà(1/16)œÅg¬∑H_s¬≤¬∑B¬∑‚àö(B/L)`, head-sea factor); `ŒîP_env=(R_AA+R_AW)¬∑STW/Œ∑`;
   `P_shaft = P_corrected + ŒîP_env`.
7. `me_foc = P_shaft¬∑SFOC¬∑h/1e6` (**C2**), SFOC load-dependent + a per-vessel
   **secular engine drift** `(1 + sfoc_drift_frac)` (reset at
   `engine_overhaul ‚à™ dry_dock`, capped +6%) + engine-degradation anomaly. The
   drift is baked into **both** truth `sfoc_g_kwh` and `me_foc`, so the C2 energy
   balance is preserved. + aux/boiler ‚Üí `total_foc`.
8. RPM from STW & target slip; `slip=(V_th‚àíSTW)/V_th ‚àà [‚àí5%,30%]` (**C7**).
9. `SOG = STW + current projection` (**C4**); `distance = speed¬∑h` (**C3**).
10. `co2 = Œ£ FOC¬∑C_F` (**C8**), carbon factors below.
11. **Noise last** ‚Äî bounded sensor noise on measured fields + labeled anomalies;
    distance/SOG re-derived from noised speeds; truth kept un-noised (**C12**,
    **C13** input). UWI dates map current true `s` ‚Üí hull fouling rating/coverage
    (**C10**); `propeller_condition` (Rubert A‚ÄìF) is banded from
    `propeller_roughness_um` and `coating_condition` from `coating_breakdown_pct`.

Propeller roughness and coating breakdown are **independent resettable
processes** ‚Äî their own reset clocks, **not** derived from hull fouling `s`.
`propeller_roughness_um` grows 150‚Üí600 ¬µm at `U(0.35,0.75)` ¬µm/day since the last
`propeller_polishing ‚à™ dry_dock` (then Rubert-banded into `propeller_condition`);
`coating_breakdown_pct` grows 0‚Äì100 % at `U(0.02,0.05)` %/day since the last
`coating_renewal ‚à™ dry_dock`. The SFOC secular drift (step 7) resets at
`engine_overhaul ‚à™ dry_dock`.

**Consistency validator** (`validate.py`, run via CLI `validate` or
`generate --validate`) checks each rule with a per-rule tolerance and prints
PASS/FAIL + violation counts (nonzero exit on fail):

| rule | check | tolerance |
|---|---|---|
| C1 | shaft power = clean-curve power(V_ref) + env power | ‚â§5% |
| C2 | implied SFOC = me_foc¬∑1e6/(power¬∑h) vs truth | ‚â§5% |
| C3 | distance = speed¬∑hours | ‚â§1% |
| C4 | SOG ‚àí STW = current projection | ‚â§0.05 kn |
| C5 | displacement = hydrostatic(mean_draft) | ‚â§2% |
| C6 | mean=(fore+aft)/2, trim=aft‚àífore | 1e-6 m |
| C7 | slip ‚àà [‚àí5%, 30%] | band |
| C8 | co2 from total_foc = co2 from component sum; factor known | ‚â§0.5% |
| C9 | per-segment slope ‚â• 0; step-down at reset events | slope ‚â• ‚àí3e-5 |
| C10 | Spearman(hull_fouling_rating, true s) | ‚â•0.7 |
| C11 | Pearson(beaufort, wave/wind) ‚â•0.6; corr(temp, density) ‚â§ ‚àí0.6 | ‚Äî |
| C12 | non-anomaly measured within noise bound; gross outliers all labeled | ‚â§3% |
| C13 | closed-loop ISO speed-loss recovery | **deferred to M2** |
| C15 | per-segment `propeller_roughness_um` non-decreasing; Spearman(days-since-polish, roughness) | ‚â•0.7 |
| C16 | `coating_breakdown_pct` non-decreasing within cycle; Spearman(days-since-coating-reset, breakdown) | ‚â•0.7 |
| C17 | load-normalized SFOC Spearman(days-since-overhaul) per engine cycle ‚â•0.5; median SFOC ‚àà [160,215] | band |

Physics rules (C1‚ÄìC7, C12) evaluate steady **at-sea, non-anomaly** points;
labeled anomalies are expected outliers and are exercised separately by C12.

### Anomalies (labeled, for M3)
`engine_degradation` (persistent SFOC step ‚Äî FOC up, speed loss flat),
`propeller` (temporary slip jump), `weather` (isolated rough days), `sensor`
(single-field gross outliers). Every outlier carries `anomaly_cause` /
`anomaly_severity` in ground truth. The M3 detector first **detrends the secular
SFOC drift per engine cycle** (Theil-Sen of load-normalized SFOC vs
days-since-overhaul) so the discrete `engine_degradation` step still stands out
against the smooth drift.

### Determinism
One master `--seed` ‚Üí `numpy.random.SeedSequence([seed, imo, purpose])`
substreams keyed by `(imo, purpose)` (fouling / uwi / environment / operating /
noise / anomaly / fuel_price / prop_rough / coating / engine / **route**). The
prop_rough / coating / engine streams are the independent propeller-roughness,
coating-breakdown and engine-drift/overhaul processes; **`route` (purpose id 11)**
is a dedicated substream whose *only* draw is the port-rotation start index (¬ß9),
so `operating` stays reserved for physics-relevant draws. Order-independent:
adding a field or vessel never shifts another vessel's draws; global `np.random`
is never touched.

> **Seed-42 rebuild (Phase 1).** Reworking the operating profile into a per-fleet
> port rotation (¬ß9) shifted the `operating` substream's draw sequence, so the
> `--seed 42` dataset is **new** (new `sfoc_base` / `base_slip` per vessel). The
> acceptance gate is unchanged: **C1‚ÄìC17 stay green + the new C18 voyage energy
> balance passes** (curated-dataset ¬ß7). The fouling / env / noise / anomaly
> substreams are untouched ‚Äî only positions, ports, `voyage_no`, `leg`, and the
> physics that keys off the reshuffled `operating` draws move.

## 6. YM WELLNESS storyline (`9700006`)

An engineered fouling arc (parameter override in `fouling.py`): an early dry dock
resets the hull, then a **long elevated segment** is seeded to breach the
~**10% maintenance trigger** (code constant `MT_TRIGGER_PCT=8.0`; the ~10% is
narrative), corroborated by rising UWI `hull_fouling_rating`; a hull cleaning
then **recovers** it (step-down to ~0). This is the "gradual rise ‚Üí cross
threshold ‚Üí cleaned ‚Üí recovered" narrative for the Dashboard deep-dive.

### 9.2 Great-circle geometry

Pure-`math` helpers, Earth radius **3440.065 nm**:

- `haversine(lat1, lon1, lat2, lon2) ‚Üí nm` ‚Äî great-circle distance.
- `gc_point(lat1, lon1, lat2, lon2, frac)` ‚Äî point `frac` (0‚Üí1) along the great
  circle via **3-D unit-vector slerp** (spherical linear interpolation). It is
  **antimeridian-safe** ‚Äî it never linearly interpolates longitude.
- `route_path(a, b) = [A, *waypoints, B]` ‚Äî the ordered lat/lon path for a leg
  (waypoints oriented A‚ÜíB).
- `path_distance_nm(path)` ‚Äî Œ£ great-circle distance over consecutive points.
- `path_point(path, frac)` ‚Äî walks the multi-segment path by **cumulative
  great-circle distance** (slerp within the active segment).

### 9.3 Route waypoints (`ROUTE_WAYPOINTS`)

Ocean waypoints bend a leg around land so map arcs stay over water, keyed by the
undirected port pair; `route_path` reverses them for the opposite direction. Any
pair **not** listed uses the direct great circle.

- **AEDXB ‚Üî {DEHAM, NLRTM}** threads Bab-el-Mandeb `(12.6, 43.4)` ‚Üí Suez
  `(30.0, 32.6)` ‚Üí Port Said `(31.5, 32.3)` ‚Üí Gibraltar `(35.95, ‚àí5.6)` ‚Üí Ushant
  `(48.5, ‚àí5.5)`.
- **SGSIN ‚Üî LKCMB** pulls north through one Malacca/Andaman point `(5.5, 95.0)`.

### 9.4 Per-fleet rotations (`_ROTATIONS`, in `generate.py`)

`build_operating_profile` walks the vessel's fleet rotation **one leg at a time**
(replacing the old random-heading dead-reckoning walk). Each rotation is an
**out-and-back loop**, so the ballast return leg retraces the laden leg rather
than arcing over land; `laden` / `ballast` alternate by leg parity.

| Fleet | Rotation | Shape |
|---|---|---|
| **FL-IA** Intra-Asia | CNSHA ¬∑ KRPUS ¬∑ JPTYO ¬∑ HKHKG ¬∑ SGSIN | loop; **never** calls a European port |
| **FL-TP** Trans-Pacific | CNSHA ¬∑ KRPUS ¬∑ USLAX ¬∑ JPTYO | pendulum, returns via Tokyo |
| **FL-AE** Asia-Europe | CNSHA ¬∑ SGSIN ¬∑ LKCMB ¬∑ AEDXB ¬∑ DEHAM ¬∑ NLRTM ¬∑ AEDXB ¬∑ LKCMB ¬∑ SGSIN | pendulum via Suez |

Per leg: the ports, path and positions are **derived** from the great circle;
`leg_days = max(2, round(path_distance_nm / (nominal ¬∑ 24)))` with
`nominal = max(6, design_speed ¬∑ speed_factor)`. At-sea positions come from
`path_point`; the single in-port day is pinned at the destination port's
coordinates. The **only** `rng_route` draw is the rotation **start index** (so
vessels spread around the loop); `rng_op` keeps its physics draws per leg
(`heading`, `speed_factor`, loading √ó3, then one speed per steaming day).

### 9.5 KEY INVARIANT ‚Äî positions are decorative

`heading_deg` stays a random `rng_op` draw ‚Äî **not** the great-circle bearing ‚Äî so
geography can **never** leak into the physics. Fuel, CII, ISO speed loss and the
C3 distance identity all key off **distance & speed, never lat/lon**. The
lat/lon/ports/`voyage_no` ride through raw ‚Üí curated ‚Üí API ‚Üí map purely to draw
the Fleet Map and per-vessel track; they carry no physical meaning. This is why
adding them left C1‚ÄìC17 green (only the reshuffled `operating` draws moved the
dataset; see the seed-42 note in ¬ß5).
